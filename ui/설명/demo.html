한국어 비속어 필터 API
분리 배포할 FE 방향을 보여주는 아주 작은 정적 데모입니다.

빠르고 가벼운 API 포털

기존 README와 정적 테스트 페이지를 제품형 FE로 분리한다는 가정의 데모입니다. 실제 구현은 Vite + React + TypeScript 기반으로 만들고, k3s에는 정적 파일 컨테이너로 배포하는 구성이 적절합니다.

Vite

빠른 개발과 단순 배포

개발 서버가 빠르고, 배포 결과물은 정적 파일이라 Nginx 컨테이너로 서빙하기 쉽습니다.

React

AI 친화적인 UI 구성

폼, 콘솔, 응답 뷰어, 문의 화면을 컴포넌트로 나누기 좋고 AI 생성 예시가 많습니다.

shadcn/ui

예쁜 기본값과 수정 가능성

컴포넌트 코드가 프로젝트에 들어오므로 원하는 디자인으로 직접 고치기 쉽습니다.

이 HTML은 실제 Vite/React 앱이 아닙니다. 앞으로 만들 화면의 정보 구조를 보여주는 목업입니다.

API Key 발급

사용자가 직접 이름, 이메일, 사용 목적을 입력하고 API key를 발급받는 흐름입니다.

테스트 콘솔

기존 `index.html`을 제품형 콘솔로 바꾸면 이런 형태가 됩니다.

단어 제안

신규 단어 추가, 오탐 제거, 기존 단어 수정 요청을 받는 화면입니다.
